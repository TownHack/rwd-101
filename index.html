Wait... Responsive what?

Responsive Web Design: The practice of designing flexible layouts that do not conform to a specific device's orientation or screen size. [Example]

- What is Responsive Web Design? - Not about designing for mobile - Not even about designing for desktops - It's about a better practice

Why is this important?

There are too many devices and screen sizes to predict every possible outcome

- Websites are meant to be dynamic

Some Statistics

  1. Desktop usage has dropped from about 82% to 62%
  2. Mobile usage has raised from about 13% to 31%
  3. Tablet/Console usage has had a slight increase

What does this mean?

Desktop no longer dominates the realm of web browsers

- Something needs to change - We can't keep trying to develop for each possible outcome independently

So, how do you do it?

CSS Media Queries!

- Ask if anyone knows how it is done

Media Queries allow you to test the device for certain properties...


  • Screen Width & Height
  • Device Orientation
  • & More!


...and use the results to help change the layout or style with CSS.

Simple Example

body {
  background-color: #ddd;
}

@media all and (min-width: 50rem) {
  body {
    background-color: #444;
  }
}

Screens less than 50rem wide will have a light background.

Screens 50rem or wider will have a dark background.

Example

How to use Media Queries

1. Inside a CSS file or Style block

@media all and (min-width: 50rem) {
  body {
    background-color: #444;
  }
}

2. Inside the link tag

3. In combination with the @import statement

@import url('large.css') all and (min-width: 50rem);
